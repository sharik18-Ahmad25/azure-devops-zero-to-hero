# 🧩 Key Concepts

---

## 🔹 Test Case

👉 A test case is a step-by-step process to test a feature

**Example**:

* Open login page
* Enter username/password
* Click login
* Verify success

---

## 🔹 Test Suite

👉 A group of test cases

**👉 Example**:

`Login Tests → multiple test cases`

---

## 🔹 Test Plan

👉 Collection of test suites

**👉 Think like**:

`Test Plan → Test Suites → Test Cases`

---

## 🔹 Bug

`👉 If test fails → bug is created`

---

## 🔁 Testing Flow (Very Important)

`Write Test Case → Run Test → Pass/Fail → Create Bug → Fix → Retest`

---

# 🚀 How to Use Azure Test Plans (Step-by-Step)

---

## 🔹 Step 1: Open Azure DevOps

* Go to https://aex.dev.azure.com/
* Open your project

---

## 🔹 Step 2: Open Test Plans

`Click Test Plans from left menu`

---

## 🔹 Step 3: Create Test Plan

* Click New Test Plan
* Enter name (e.g., Login Testing)
* Save

---

## 🔹 Step 4: Create Test Suite

* Inside test plan
* Click New Test Suite
* Add test cases group

---

## 🔹 Step 5: Add Test Cases

* Click New Test Case
* add steps:
 * Input
 * Expected output

 ---

## 🔹 Step 6: Run Test

* Click Run Test
* Mark result:
 * Pass ✅
 * Fail ❌

 ---

## 🔹 Step 7: Create Bug

* If test fails
* Create bug directly

**👉 Bug will be tracked in Azure Boards**
